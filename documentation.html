Overview

SegmentIQ is a single-file Python application that applies K-Means clustering to retail customer data, then serves a fully interactive analytics dashboard in your default web browser — no Flask, no Node, no configuration files required.

🧮
Smart clustering
K-Means++ initialisation with automatic K selection via the elbow method.
📊
Live dashboard
Four-tab browser dashboard with scatter plots, histograms, PCA, and profiles.
⚡
Real-time predict
Type in a customer's income, spending, and age — get their cluster instantly.
🔄
Browser re-cluster
Adjust K and iterations in the UI and re-run K-Means without restarting Python.
📁
Synthetic fallback
Works without the CSV — generates a representative 200-customer dataset.
💾
Export HTML
Saves a standalone customer_segmentation_dashboard.html you can share.

Architecture

Everything runs in a single Python file. The flow is:

1

Data Ingestion

Reads Mall_Customers.csv (or generates synthetic data). Normalises column names automatically.

2

Preprocessing

Features are standardised with StandardScaler (zero mean, unit variance) before clustering.

3

Elbow + Silhouette

Runs K-Means for K=2..10, computes WCSS and silhouette scores, picks the elbow automatically.

4

K-Means Fit

Final model fitted with k-means++ init, 15 random restarts, up to 300 iterations.

5

HTML Generation

All data, cluster labels, centroids, PCA projections, and profiles are serialised into a self-contained HTML file.

6

Serve & Open

A lightweight http.server thread serves the page, then webbrowser.open() launches your browser.

Quickstart

1 — Install dependencies

# Only 3 packages needed
pip install pandas numpy scikit-learn

2 — Run (without the CSV)

python kmeans_customer_segmentation.py

A browser window opens automatically at http://localhost:8765 with 200 synthetic customers pre-loaded.

3 — Run with the Kaggle dataset

Download Mall_Customers.csv from Kaggle, then:

python kmeans_customer_segmentation.py --data Mall_Customers.csv

4 — Force a specific K

python kmeans_customer_segmentation.py --data Mall_Customers.csv --k 5
✓
Standalone HTMLA file named customer_segmentation_dashboard.html is saved in your working directory every time you run the script. You can open this file directly in any browser — no Python needed.

K-Means Algorithm

K-Means partitions n customers into k non-overlapping groups by minimising the within-cluster sum of squares (WCSS): 

WCSS = Σ Σ ||x_i − μ_k||²
      k  x_i∈C_k

where μ_k is the centroid (mean) of cluster C_k.

k-means++ Initialisation

Vanilla K-Means is sensitive to centroid initialisation. k-means++ selects initial centroids with probability proportional to their squared distance from already-chosen centroids, dramatically reducing variance and converging faster.

Algorithm Steps

1

Standardise features

All features are scaled to zero mean and unit variance so income (range ~$15k–$137k) doesn't dominate spending score (1–100).

2

Initialise centroids (k-means++)

First centroid chosen at random. Each subsequent centroid chosen with probability proportional to D(x)² — the squared distance to the nearest existing centroid.

3

Assignment step

Each customer is assigned to the cluster whose centroid minimises Euclidean distance in the standardised feature space.

4

Update step

Each centroid is recomputed as the mean of all customers assigned to it.

5

Convergence check

Steps 3–4 repeat until centroid positions change by less than 1×10⁻⁹ or max_iter is reached.

Elbow Method & Auto-K

The elbow method selects K by looking for the "elbow" — the point where adding more clusters yields diminishing WCSS improvements. The script automates this using the second derivative of the WCSS curve: 

def auto_k(wcss):
    d2 = np.diff(wcss, 2)   # second finite difference
    return int(np.argmax(d2) + 3)   # +3 accounts for offset

The elbow chart in the dashboard highlights the selected K in red. You can override it with --k or use the K slider in the Clusters tab to re-run interactively.

Silhouette Score

The silhouette score measures how well-separated the clusters are, ranging from −1 (wrong cluster) to +1 (perfect separation). Values above 0.5 indicate good clustering.

s(i) = (b(i) − a(i)) / max(a(i), b(i))

a(i) = mean distance to points in same cluster
b(i) = mean distance to points in nearest other cluster
⚠
Typical values for this datasetWith K=5 on the Kaggle dataset you should see a silhouette score around 0.55–0.57. Scores below 0.4 suggest K is too high or too low.

PCA Projection

Principal Component Analysis (PCA) projects the standardised feature space into 2D for visualisation. The two principal components are shown in the Insights → PCA 2D Projection chart, with the percentage of variance explained noted on each axis. 

from sklearn.decomposition import PCA
pca  = PCA(n_components=2).fit(X_scaled)
proj = pca.transform(X_scaled)   # shape: (n_customers, 2)
var  = pca.explained_variance_ratio_

Dashboard Tabs

Overview

  • Metric cards — total customers, average income, spending score, and age.
  • Income histogram — frequency distribution of annual income across all customers.
  • Spending score histogram — distribution of spending scores.
  • Raw scatter — income vs spending, uncoloured, before any clustering is applied.

Clusters

  • K slider — adjust K from 2 to 8 and click "Re-run K-Means" to recompute in the browser.
  • Elbow curve — WCSS vs K with the auto-selected K highlighted.
  • Cluster map — scatter plot coloured by cluster, with star markers for centroids.
  • Profiles table — per-cluster averages for income, spending, age, and size.

Predict Tab

Enter a customer's age, annual income, and spending score and click Classify Customer. The dashboard will:

  1. Standardise the inputs using the same scaler parameters as the training data.
  2. Compute Euclidean distance from the input to each cluster centroid in standardised space.
  3. Assign the customer to the nearest centroid.
  4. Display the cluster name, a confidence estimate, cluster averages, and a marketing strategy recommendation.
  5. Plot a ★ star on the cluster map at the customer's position.
ℹ
Confidence scoreThe confidence is a heuristic: max(0, 100 − dist² × 12). It drops as the customer moves further from the centroid. It is not a probabilistic estimate.

Re-clustering in the browser

The Clusters tab includes a full JavaScript K-Means implementation that runs entirely in your browser. Adjust K and iterations, then click Re-run K-Means — no round-trip to Python needed. The implementation uses k-means++ initialisation matching the Python version.

Insights Tab

  • Cluster sizes — bar chart showing how many customers are in each cluster.
  • Spending by cluster — average spending score per cluster.
  • PCA 2D projection — clusters visualised in reduced 2D space.
  • Marketing recommendations — one card per cluster with icon, segment name, averages, and a targeted marketing strategy.

Segment Profiles

Segments are named heuristically based on median income and spending score. With the default K=5 on the Kaggle dataset, you will typically see:

IconNameIncomeSpendingStrategy
💎High RollersHighHighVIP perks, exclusive access, white-glove service
🏦Affluent SaversHighLowAspirational upsell, prestige product showcases
🛍️Impulsive BuyersLowHighBNPL, scarcity messaging, limited-time offers
🪙Budget ShoppersLowLowValue bundles, cashback, referral incentives
👥Mainstream CrowdMidMidPersonalised recommendations, loyalty points

The naming logic (in make_profiles) compares each cluster's average income and spending against the dataset median. Clusters above both medians → High Rollers; above income only → Affluent Savers; etc.

Troubleshooting

Browser doesn't open automatically

Open http://localhost:8765 manually, or use --port to try a different port. On some Linux systems webbrowser.open requires a desktop environment.

Port already in use

python kmeans_customer_segmentation.py --port 9000

CSV columns not recognised

The script normalises column names case-insensitively. If your CSV has unusual names, check that they contain the substrings income, spend, age, or gender.

ModuleNotFoundError

pip install pandas numpy scikit-learn

No other packages are needed. Chart.js is loaded from a CDN in the browser.

Charts not rendering

The dashboard loads Chart.js from cdn.jsdelivr.net. If you're offline, download chart.umd.min.js locally and update the <script src> tag in the HTML template.

ℹ
Headless environmentsIf running on a remote server without a display, the script still works — it saves the HTML file and starts the server. Use SSH port-forwarding: ssh -L 8765:localhost:8765 user@server, then open http://localhost:8765 locally.
